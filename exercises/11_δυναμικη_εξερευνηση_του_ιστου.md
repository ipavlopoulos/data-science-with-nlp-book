# Ασκήσεις κεφαλαίου 11

## Ασκήσεις κατανόησης

1. Ποια είναι η διαφορά ανάμεσα σε scraping μιας σελίδας και crawling πολλών σελίδων;
2. Γιατί χρειάζονται περιορισμοί βάθους και πλήθους σελίδων σε έναν crawler;
3. Ποια προβλήματα προσπαθεί να αποφύγει η χρήση `visited` συνόλου;

## Προγραμματιστικές ασκήσεις

1. Υλοποιήστε απλό BFS crawler που ξεκινά από ένα URL και συλλέγει εσωτερικούς συνδέσμους.
2. Προσθέστε όριο μέγιστου βάθους και μέγιστου πλήθους σελίδων.
3. Καταγράψτε για κάθε σελίδα το URL, το βάθος και τον αριθμό εξερχόμενων links.

## Ερωτήσεις πολλαπλής επιλογής

Σε κάθε ερώτηση μπορεί να υπάρχουν μία ή περισσότερες σωστές απαντήσεις.

1. Τι κάνει ένας crawler;
   - A. Μπορεί να επισκεφθεί πολλές σελίδες.
   - B. Ακολουθεί συνδέσμους.
   - C. Εγγυάται ότι η συλλογή είναι δεοντολογικά αποδεκτή.
   - D. Λειτουργεί καλύτερα όταν δεν έχει κανένα όριο.
2. Γιατί χρησιμοποιούμε σύνολο `visited`;
   - A. Για αποφυγή διπλής επεξεργασίας.
   - B. Για να ξέρουμε ποιες σελίδες έχουμε ήδη δει.
   - C. Για να αυξήσουμε απεριόριστα το βάθος.
   - D. Για αποφυγή κύκλων.
3. Πότε προτιμάται API αντί για crawling;
   - A. Όταν έχει σαφείς όρους χρήσης.
   - B. Όταν μειώνει την επιβάρυνση του server.
   - C. Όταν θέλουμε να σεβαστούμε rate limits και κανόνες χρήσης.
   - D. Όταν παρέχει δομημένα δεδομένα.

## Εργασία επέκτασης

- Μελετήστε ένα δημόσιο API και συγκρίνετε τη συλλογή δεδομένων μέσω API με τη συλλογή μέσω crawling.
