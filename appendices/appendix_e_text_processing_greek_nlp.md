# Παράρτημα Ε: Επεξεργασία (ελληνικού) κειμένου

Η επεξεργασία φυσικής γλώσσας μετατρέπει το κείμενο σε μορφές που μπορούν να αναλυθούν υπολογιστικά. Στο βιβλίο, το κείμενο εμφανίζεται σε πολλές μορφές: παροιμίες, ντοπιολαλιά, σχόλια, επισημειώσεις, γλωσσικά μοντέλα και αναπαραστάσεις κειμένου.

Το παράρτημα αυτό συγκεντρώνει βασικές έννοιες επεξεργασίας κειμένου, με ιδιαίτερη έμφαση σε ζητήματα που εμφανίζονται συχνά στα ελληνικά.

## Ε.1 Κείμενο ως δεδομένα

Το κείμενο δεν είναι απλώς μια ακολουθία χαρακτήρων. Περιέχει λέξεις, σημεία στίξης, μορφολογικές παραλλαγές, ορθογραφικές επιλογές, τόνους, κεφαλαία και πεζά, καθώς και πληροφορίες συμφραζομένων.

Για να χρησιμοποιηθεί σε ανάλυση δεδομένων ή σε μοντέλα μηχανικής μάθησης, χρειάζεται να μετατραπεί σε πιο δομημένη μορφή.

Συνηθισμένα βήματα είναι:

- καθαρισμός,
- κανονικοποίηση,
- τμηματοποίηση,
- εξαγωγή χαρακτηριστικών,
- διανυσματική αναπαράσταση,
- μοντελοποίηση.

Δεν εφαρμόζονται πάντα όλα τα βήματα. Η κατάλληλη επεξεργασία εξαρτάται από το ερώτημα.

## Ε.2 Κωδικοποίηση και Unicode

Τα ελληνικά κείμενα πρέπει να αποθηκεύονται και να διαβάζονται με σωστή κωδικοποίηση χαρακτήρων. Η σύγχρονη προεπιλογή είναι συνήθως το UTF-8.

Προβλήματα κωδικοποίησης μπορεί να εμφανιστούν ως αλλοιωμένοι χαρακτήρες, για παράδειγμα όταν ένα αρχείο διαβάζεται με λάθος encoding.

Με τη βιβλιοθήκη pandas, μπορούμε να ορίσουμε κωδικοποίηση κατά την ανάγνωση:

```python
import pandas as pd

df = pd.read_csv("data/texts.csv", encoding="utf-8")
```

Αν ένα αρχείο δεν διαβάζεται σωστά, χρειάζεται να ελεγχθεί με ποια κωδικοποίηση δημιουργήθηκε.

## Ε.3 Πεζοποίηση και κεφαλαία

Η πεζοποίηση μειώνει τις παραλλαγές που οφείλονται μόνο σε κεφαλαία και πεζά:

```python
text = "Καλή Μέρα"
text.lower()
```

Αποτέλεσμα:

```text
καλή μέρα
```

Αυτό είναι χρήσιμο όταν δεν μας ενδιαφέρει αν μια λέξη γράφτηκε με κεφαλαίο. Δεν είναι πάντα ουδέτερη επιλογή: σε ονόματα, ακρωνύμια ή ειδικές γραφές, τα κεφαλαία μπορεί να μεταφέρουν πληροφορία.

## Ε.4 Τόνοι και διακριτικά

Στα ελληνικά, οι τόνοι και άλλα διακριτικά μπορεί να επηρεάζουν την αναζήτηση και την αντιστοίχιση λέξεων.

Παράδειγμα:

```text
άλλα
αλλά
```

Οι δύο λέξεις έχουν διαφορετική σημασία. Αν αφαιρέσουμε τους τόνους, γίνονται ίδιες:

```text
αλλα
```

Η αφαίρεση τόνων μπορεί να είναι χρήσιμη για αναζήτηση ή αντιστοίχιση παραλλαγών, αλλά μπορεί να προκαλέσει απώλεια πληροφορίας. Για τον λόγο αυτό, είναι συχνά καλή πρακτική να κρατάμε και την αρχική και την κανονικοποιημένη μορφή.

Παράδειγμα αφαίρεσης τόνων:

```python
import unicodedata

def remove_accents(text):
    decomposed = unicodedata.normalize("NFD", text)
    return "".join(
        ch for ch in decomposed
        if unicodedata.category(ch) != "Mn"
    )

remove_accents("καλή μέρα")
```

## Ε.5 Τελικό σίγμα

Το ελληνικό σίγμα έχει δύο πεζές μορφές:

```text
σ  μέσα στη λέξη
ς  στο τέλος της λέξης
```

Παράδειγμα:

```text
κόσμος
```

Σε ορισμένες εφαρμογές, μπορεί να είναι χρήσιμο να ενοποιηθεί το τελικό σίγμα με το απλό σίγμα:

```python
text = text.replace("ς", "σ")
```

Αυτό διευκολύνει την αντιστοίχιση χαρακτήρων και n-grams, αλλά αλλοιώνει την κανονική ορθογραφική μορφή. Γι' αυτό η επιλογή πρέπει να τεκμηριώνεται.

## Ε.6 Σημεία στίξης, αριθμοί και κενά

Τα σημεία στίξης και τα κενά μπορεί να δημιουργούν τεχνικές παραλλαγές:

```text
καλή μέρα
καλή   μέρα
καλή μέρα!
```

Συνηθισμένες επιλογές καθαρισμού:

- αφαίρεση περιττών κενών,
- ενοποίηση πολλαπλών κενών σε ένα,
- αφαίρεση ή διατήρηση στίξης,
- χειρισμός αριθμών,
- χειρισμός URLs και mentions.

Παράδειγμα ενοποίησης κενών:

```python
import re

text = re.sub(r"\s+", " ", text).strip()
```

Η στίξη δεν πρέπει να αφαιρείται μηχανικά σε όλες τις εφαρμογές. Σε συναισθηματική ανάλυση, για παράδειγμα, τα θαυμαστικά μπορεί να είναι πληροφοριακά.

## Ε.7 Τμηματοποίηση

Η τμηματοποίηση χωρίζει το κείμενο σε μονάδες. Οι μονάδες μπορεί να είναι:

- λέξεις,
- χαρακτήρες,
- υπολέξεις,
- προτάσεις.

Απλή τμηματοποίηση με κενά:

```python
text = "η καλή μέρα φαίνεται"
tokens = text.split()
```

Αποτέλεσμα:

```text
["η", "καλή", "μέρα", "φαίνεται"]
```

Η απλή τμηματοποίηση δεν χειρίζεται πάντα σωστά στίξη, αποστρόφους, συντομογραφίες ή ειδικές μορφές. Για πιο απαιτητική επεξεργασία, χρησιμοποιούνται ειδικά εργαλεία NLP.

## Ε.8 Λέξεις, χαρακτήρες και υπολέξεις

Η επιλογή μονάδας επηρεάζει την ανάλυση.

### Λέξεις

Τα word tokens είναι εύκολα στην ερμηνεία:

```text
["καλή", "μέρα"]
```

Μειονέκτημα: διαφορετικές μορφές της ίδιας λέξης αντιμετωπίζονται ως διαφορετικές.

### Χαρακτήρες

Τα character n-grams είναι χρήσιμα όταν ενδιαφέρουν ορθογραφικά ή μορφολογικά μοτίβα:

```text
καλ, αλή, λή_
```

Στα ελληνικά, είναι ιδιαίτερα χρήσιμα για ντοπιολαλιά, μορφολογικές παραλλαγές και παροιμιακό λόγο.

### Υπολέξεις

Οι υπολέξεις βρίσκονται ανάμεσα στις λέξεις και στους χαρακτήρες. Χρησιμοποιούνται συχνά σε νεότερα γλωσσικά μοντέλα, επειδή μπορούν να χειριστούν άγνωστες ή σπάνιες λέξεις.

## Ε.9 Ν-γράμματα (n-grams)

Ένα n-gram είναι μία ακολουθία από n διαδοχικές μονάδες (όπως λέξεις ή χαρακτήρες).

Παράδειγμα με λέξεις:

```text
κείμενο: η καλή μέρα φαίνεται
unigrams: η, καλή, μέρα, φαίνεται
bigrams: η καλή, καλή μέρα, μέρα φαίνεται
trigrams: η καλή μέρα, καλή μέρα φαίνεται
```

Παράδειγμα με χαρακτήρες:

```text
λέξη: μέρα
character bigrams: μέ, έρ, ρα
character trigrams: μέρ, έρα
```

Τα n-grams είναι απλά, διαφανή και συχνά πολύ ισχυρά χαρακτηριστικά για ταξινόμηση κειμένων.

## Ε.10 Stopwords

Τα stopwords είναι πολύ συχνές λέξεις που συχνά αφαιρούνται, όπως άρθρα, σύνδεσμοι ή αντωνυμίες.

Παράδειγμα:

```text
και, να, το, η, ο
```

Η αφαίρεση stopwords μπορεί να βοηθήσει όταν μας ενδιαφέρουν θεματικές λέξεις. Όμως δεν είναι πάντα σωστή επιλογή. Σε γλωσσική μοντελοποίηση, στυλομετρία ή ντοπιολαλιά, οι πολύ συχνές λέξεις μπορεί να είναι πληροφοριακές.

## Ε.11 Stemming και λημματοποίηση

Το stemming κόβει καταλήξεις για να προσεγγίσει ένα κοινό στέλεχος.

Η λημματοποίηση προσπαθεί να επιστρέψει τη λεξική μορφή μιας λέξης.

Παράδειγμα:

```text
γράφω, γράφεις, γράφει → γράφω
```

Στα ελληνικά, η λημματοποίηση είναι δυσκολότερη από ό,τι σε γλώσσες με απλούστερη μορφολογία, επειδή οι λέξεις κλίνονται πλούσια ως προς πτώση, γένος, αριθμό, πρόσωπο, χρόνο και έγκλιση.

Η επιλογή ανάμεσα σε αρχικές λέξεις, stems και λήμματα εξαρτάται από την εφαρμογή.

## Ε.12 Μορφολογία ελληνικών

Η ελληνική γλώσσα έχει πλούσια μορφολογία. Η ίδια λέξη μπορεί να εμφανιστεί σε πολλές μορφές:

```text
καλός, καλή, καλό, καλοί, καλών
```

Αυτό επηρεάζει:

- το μέγεθος του λεξιλογίου,
- τη συχνότητα κάθε μορφής,
- την αραιότητα των χαρακτηριστικών,
- την απόδοση μοντέλων που βασίζονται σε λέξεις.

Τα character n-grams και τα υπολεξικά μοντέλα, όπως το fastText, μπορούν να βοηθήσουν επειδή αξιοποιούν πληροφορία μέσα στη λέξη.

## Ε.13 Ντοπιολαλιά και παραλλαγές

Σε δεδομένα ντοπιολαλιάς, η γλωσσική ποικιλία είναι μέρος του σήματος που θέλουμε να μελετήσουμε.

Παραλλαγές μπορεί να εμφανίζονται:

- στην ορθογραφία,
- στη φωνολογική απόδοση,
- στη μορφολογία,
- στο λεξιλόγιο,
- στη χρήση τοπικών εκφράσεων.

Σε τέτοια δεδομένα, ο υπερβολικός καθαρισμός μπορεί να αφαιρέσει χρήσιμη πληροφορία. Αν το ζητούμενο είναι ο γεωεντοπισμός ή η ανάλυση ντοπιολαλιάς, τότε οι παραλλαγές αυτές συχνά πρέπει να διατηρούνται.

## Ε.14 Παροιμίες ως γλωσσικά δεδομένα

Οι παροιμίες έχουν ιδιαίτερα χαρακτηριστικά:

- είναι σύντομες,
- συχνά έχουν παγιωμένες μορφές,
- μπορεί να εμφανίζονται με παραλλαγές,
- συνδέονται με γεωγραφικές περιοχές,
- περιέχουν μεταφορική γλώσσα,
- μπορεί να έχουν παλαιότερες ή τοπικές μορφές.

Αυτό σημαίνει ότι η απλή αντιστοίχιση λέξεων δεν αρκεί πάντα. Χρειάζονται μέθοδοι που χειρίζονται παραλλαγές, ομοιότητα και γεωγραφική πληροφορία.

## Ε.15 Bag-of-words

Το bag-of-words αναπαριστά ένα κείμενο με βάση τις λέξεις που περιέχει, αγνοώντας τη σειρά τους.

Παράδειγμα:

```text
κείμενο: η καλή μέρα
χαρακτηριστικά: η=1, καλή=1, μέρα=1
```

Πλεονέκτημα: είναι απλό και ερμηνεύσιμο.

Μειονέκτημα: χάνει τη σειρά των λέξεων και τα συμφραζόμενα.

## Ε.16 TF-IDF και χαρακτηριστικές λέξεις

Το TF-IDF δίνει βάρος σε λέξεις ή n-grams που είναι συχνά σε ένα κείμενο ή κατηγορία, αλλά όχι παντού στο σύνολο δεδομένων.

Χρήσιμη διαίσθηση:

- μια λέξη που εμφανίζεται συχνά σε μια περιοχή είναι σημαντική,
- αλλά αν εμφανίζεται συχνά σε όλες τις περιοχές, δεν είναι τόσο διακριτική.

Το TF-IDF είναι χρήσιμο για:

- αναζήτηση,
- ταξινόμηση,
- εύρεση χαρακτηριστικών όρων,
- σύγκριση περιοχών ή κατηγοριών.

## Ε.17 Διανυσματικές αναπαραστάσεις

Οι διανυσματικές αναπαραστάσεις μετατρέπουν λέξεις, προτάσεις ή κείμενα σε διανύσματα αριθμών.

Σε αντίθεση με το bag-of-words, οι πυκνές αναπαραστάσεις μπορούν να αποτυπώνουν σημασιολογικές σχέσεις:

```text
ομοιότητα(σκύλος, γάτα) > ομοιότητα(σκύλος, τραπέζι)
```

Τέτοιες αναπαραστάσεις χρησιμοποιούνται για:

- σημασιολογική ομοιότητα,
- αναζήτηση παρόμοιων κειμένων,
- ταξινόμηση,
- ομαδοποίηση,
- οπτικοποίηση.

## Ε.18 fastText και υπολεξική πληροφορία

Το fastText μαθαίνει αναπαραστάσεις λέξεων αξιοποιώντας και υπολεξικά n-grams. Αυτό είναι χρήσιμο για γλώσσες με πλούσια μορφολογία, όπως τα ελληνικά.

Παράδειγμα:

```text
γράφω, γράφεις, γράφει
```

Οι λέξεις αυτές μοιράζονται χαρακτήρες και μορφολογικά μοτίβα. Ένα υπολεξικό μοντέλο μπορεί να αξιοποιήσει αυτή τη συγγένεια καλύτερα από ένα μοντέλο που βλέπει κάθε λέξη ως ανεξάρτητη μονάδα.

## Ε.19 Γλωσσικά μοντέλα

Ένα γλωσσικό μοντέλο αποδίδει πιθανότητες σε ακολουθίες γλωσσικών μονάδων.

Σε ένα απλό word bigram μοντέλο:

```text
P(μέρα | καλή)
```

είναι η πιθανότητα να εμφανιστεί η λέξη μέρα μετά τη λέξη καλή.

Σε character n-gram μοντέλο, οι μονάδες είναι χαρακτήρες αντί για λέξεις. Αυτό μπορεί να είναι χρήσιμο για ντοπιολαλιά, επειδή μικρά ορθογραφικά και μορφολογικά μοτίβα μεταφέρουν τοπική πληροφορία.

## Ε.20 Ποιότητα δεδομένων κειμένου

Πριν από την ανάλυση, είναι χρήσιμο να ελέγχουμε για:

- κενές τιμές,
- διπλότυπα,
- πολύ μικρά ή πολύ μεγάλα κείμενα,
- ακραίες μορφές,
- αλλαγές κωδικοποίησης,
- ασυνέπειες σε τόνους και πεζά,
- μη ελληνικούς χαρακτήρες,
- URLs, mentions ή υπολείμματα HTML.

Παράδειγμα ελέγχου μήκους:

```python
df["char_count"] = df["text"].str.len()
df["word_count"] = df["text"].str.split().str.len()
```

## Ε.21 Ελληνικό NLP: εργαλεία και πόροι

Για ελληνικά κείμενα, μπορεί να χρειαστούν εργαλεία που υποστηρίζουν ειδικά την ελληνική γλώσσα.

Ενδεικτικές κατηγορίες πόρων:

- εργαλεία tokenization,
- εργαλεία POS tagging,
- λημματοποιητές,
- σώματα ελληνικών κειμένων,
- μοντέλα λέξεων ή προτάσεων,
- λεξικά και γλωσσικοί πόροι,
- datasets επισημείωσης.

Κατά την επιλογή εργαλείου, ελέγχουμε:

- αν υποστηρίζει ελληνικά,
- σε τι είδος ελληνικών εκπαιδεύτηκε,
- ποια άδεια χρήσης έχει,
- αν ταιριάζει με το είδος του κειμένου που μελετάμε,
- αν χειρίζεται πολυτονικό, μονοτονικό ή μη τυπική γραφή.

## Ε.22 Δεοντολογία και άδειες

Τα κείμενα είναι δεδομένα, αλλά συχνά έχουν δημιουργούς, συμφραζόμενα και περιορισμούς χρήσης.

Πριν χρησιμοποιήσουμε ένα σώμα κειμένων, ελέγχουμε:

- την άδεια χρήσης,
- αν περιέχει προσωπικά δεδομένα,
- αν επιτρέπεται η αναδιανομή,
- αν χρειάζεται ανωνυμοποίηση,
- πώς πρέπει να παρατεθεί η πηγή,
- αν η συλλογή έγινε με δεοντολογικό τρόπο.

Σε σχόλια, κοινωνικά δίκτυα ή δεδομένα επιθετικού λόγου, η προσοχή πρέπει να είναι μεγαλύτερη, επειδή τα κείμενα μπορεί να περιέχουν προσωπικές πληροφορίες ή ευαίσθητο περιεχόμενο.

## Ε.23 Πρακτική λίστα ελέγχου

Πριν χρησιμοποιήσουμε κείμενα σε μοντέλο ή ανάλυση, ρωτάμε:

- Ποια είναι η μονάδα ανάλυσης: λέξη, χαρακτήρας, πρόταση ή κείμενο;
- Χρειάζεται πεζοποίηση;
- Θα διατηρηθούν οι τόνοι;
- Πώς θα χειριστούμε το τελικό σίγμα;
- Θα αφαιρεθεί στίξη ή αριθμοί;
- Υπάρχουν ορθογραφικές ή τοπικές παραλλαγές που πρέπει να διατηρηθούν;
- Χρειάζονται stopwords;
- Χρειάζεται λημματοποίηση ή stemming;
- Είναι καλύτερα word n-grams ή character n-grams;
- Χρειάζεται TF-IDF ή πυκνή διανυσματική αναπαράσταση;
- Υπάρχουν ζητήματα άδειας ή προσωπικών δεδομένων;
- Έχουν τεκμηριωθεί οι αποφάσεις προεπεξεργασίας;

Οι αποφάσεις επεξεργασίας κειμένου δεν είναι ουδέτερες. Μπορούν να ενισχύσουν ή να αφαιρέσουν το σήμα που μας ενδιαφέρει. Γι' αυτό πρέπει να συνδέονται με το ερευνητικό ερώτημα και να καταγράφονται καθαρά.
