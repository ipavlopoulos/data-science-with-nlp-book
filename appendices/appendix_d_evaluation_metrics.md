# Παράρτημα Δ: Μετρικές αξιολόγησης

Οι μετρικές αξιολόγησης συνοψίζουν την απόδοση ενός μοντέλου, μιας επισημείωσης ή μιας λύσης ομαδοποίησης. Η επιλογή της σωστής μετρικής είναι κρίσιμη: διαφορετικές μετρικές απαντούν σε διαφορετικές ερωτήσεις και μπορούν να οδηγήσουν σε διαφορετικά συμπεράσματα.

Το παράρτημα αυτό συγκεντρώνει τις βασικές μετρικές που εμφανίζονται στα κεφάλαια του βιβλίου.

## Δ.1 Γιατί χρειαζόμαστε πολλές μετρικές

Δεν υπάρχει μία μετρική που να είναι κατάλληλη για όλα τα προβλήματα.

Για παράδειγμα, σε ένα πρόβλημα ταξινόμησης με πολύ ανισόρροπες κατηγορίες, η απλή ακρίβεια μπορεί να είναι παραπλανητική. Αν το 95% των παραδειγμάτων ανήκει στην ίδια κατηγορία, ένα μοντέλο που προβλέπει πάντα αυτή την κατηγορία έχει 95% accuracy, αλλά δεν έχει μάθει να αναγνωρίζει τις σπάνιες κατηγορίες.

Γι' αυτό εξετάζουμε τις μετρικές σε σχέση με:

- το είδος του προβλήματος,
- την ισορροπία των κατηγοριών,
- το κόστος των διαφορετικών λαθών,
- το αν μας ενδιαφέρει η συνολική απόδοση ή η απόδοση ανά κατηγορία,
- το αν οι ετικέτες είναι βέβαιες ή υποκειμενικές.

## Δ.2 Πίνακας σύγχυσης

Ο πίνακας σύγχυσης συγκρίνει τις πραγματικές με τις προβλεπόμενες κατηγορίες.

Για δυαδική ταξινόμηση:

|                     | Προβλέφθηκε θετικό | Προβλέφθηκε αρνητικό |
|---------------------|-------------------:|---------------------:|
| Πραγματικά θετικό   |                 TP |                   FN |
| Πραγματικά αρνητικό |                 FP |                   TN |

όπου:

- TP: true positives, σωστά θετικά,
- TN: true negatives, σωστά αρνητικά,
- FP: false positives, ψευδώς θετικά,
- FN: false negatives, ψευδώς αρνητικά.

Ο πίνακας σύγχυσης είναι συχνά πιο ενημερωτικός από μια μοναδική μετρική, επειδή δείχνει ποια λάθη κάνει το μοντέλο.

## Δ.3 Accuracy

Η ακρίβεια (accuracy) είναι το ποσοστό των σωστών προβλέψεων:

```text
accuracy = (TP + TN) / (TP + TN + FP + FN)
```

Σε πολυκατηγορική ταξινόμηση:

```text
accuracy = σωστές προβλέψεις / σύνολο παραδειγμάτων
```

Η ακρίβεια είναι χρήσιμη όταν:

- οι κατηγορίες είναι σχετικά ισορροπημένες,
- όλα τα λάθη έχουν παρόμοιο κόστος,
- θέλουμε μια απλή συνολική εικόνα.

Μπορεί να είναι παραπλανητική όταν οι κατηγορίες είναι πολύ ανισόρροπες.

## Δ.4 Precision

Η ακρίβεια θετικών προβλέψεων (precision) απαντά στην ερώτηση:

> Από όσα προβλέφθηκαν ως θετικά, πόσα ήταν πράγματι θετικά;

Ο τύπος είναι:

```text
precision = TP / (TP + FP)
```

Υψηλό precision σημαίνει ότι το μοντέλο κάνει λίγα ψευδώς θετικά λάθη.

Η ακρίβεια αυτή είναι ιδιαίτερα σημαντική όταν το κόστος των ψευδώς θετικών είναι υψηλό. Για παράδειγμα, αν ένα σύστημα χαρακτηρίζει σχόλια ως επιθετικά, χαμηλό precision σημαίνει ότι πολλά μη επιθετικά σχόλια μπορεί να επισημαίνονται λανθασμένα.

## Δ.5 Recall

Η ανάκληση (recall) απαντά στην ερώτηση:

> Από όλα τα πραγματικά θετικά παραδείγματα, πόσα εντόπισε το μοντέλο;

Ο τύπος είναι:

```text
recall = TP / (TP + FN)
```

Υψηλό recall σημαίνει ότι το μοντέλο χάνει λίγα θετικά παραδείγματα.

Η ανάκληση είναι ιδιαίτερα σημαντική όταν το κόστος των ψευδώς αρνητικών είναι υψηλό. Για παράδειγμα, αν θέλουμε να εντοπίσουμε προβληματικό ή επικίνδυνο περιεχόμενο, χαμηλό recall σημαίνει ότι πολλά σχετικά παραδείγματα μένουν αδιάγνωστα.

## Δ.6 F1

Το F1 συνδυάζει precision και recall με αρμονικό μέσο:

```text
F1 = 2 * precision * recall / (precision + recall)
```

Το F1 είναι υψηλό μόνο όταν και η precision και η recall είναι υψηλές. Αν μία από τις δύο είναι χαμηλή, το F1 μειώνεται.

Χρησιμοποιείται συχνά όταν θέλουμε ισορροπία ανάμεσα στα ψευδώς θετικά και στα ψευδώς αρνητικά.

## Δ.7 Specificity και sensitivity

Το sensitivity είναι άλλο όνομα για τη recall της θετικής κατηγορίας:

```text
sensitivity = TP / (TP + FN)
```

Το specificity μετρά πόσα από τα πραγματικά αρνητικά εντοπίζονται σωστά:

```text
specificity = TN / (TN + FP)
```

Οι δύο μετρικές είναι χρήσιμες όταν μας ενδιαφέρει χωριστά η συμπεριφορά του μοντέλου σε θετικές και αρνητικές περιπτώσεις.

## Δ.8 Micro, macro και weighted μέσοι όροι

Σε πολυκατηγορική ταξινόμηση, οι μετρικές όπως precision, recall και F1 μπορούν να υπολογιστούν ανά κατηγορία. Για να πάρουμε μία συνολική τιμή, χρειαζόμαστε τον μέσο όρο.

Ο macro μέσος όρος υπολογίζει τη μετρική για κάθε κατηγορία και μετά παίρνει τον απλό μέσο όρο.

```text
macro-F1 = μέσος όρος του F1 ανά κατηγορία
```

Δίνει ίσο βάρος σε κάθε κατηγορία, ανεξάρτητα από το πόσα παραδείγματα έχει.

Είναι χρήσιμος όταν μας ενδιαφέρει η απόδοση και στις σπάνιες κατηγορίες.

Ο micro μέσος όρος αθροίζει τα TP, FP και FN από όλες τις κατηγορίες και μετά υπολογίζει τη μετρική.

Δίνει μεγαλύτερο βάρος στις συχνές κατηγορίες, επειδή αυτές έχουν περισσότερα παραδείγματα.

Ο σταθμισμένος μέσος όρος υπολογίζει τη μετρική ανά κατηγορία και μετά παίρνει μέσο όρο με βάρος το πλήθος παραδειγμάτων κάθε κατηγορίας.

Είναι ενδιάμεση επιλογή: λαμβάνει υπόψη όλες τις κατηγορίες, αλλά οι συχνότερες επηρεάζουν περισσότερο το αποτέλεσμα.

## Δ.9 Παράδειγμα ταξινόμησης

Έστω δυαδικό πρόβλημα με:

```text
TP = 40
FP = 10
FN = 20
TN = 130
```

Τότε:

```text
accuracy  = (40 + 130) / 200 = 0.85
precision = 40 / (40 + 10) = 0.80
recall    = 40 / (40 + 20) = 0.67
F1        = 2 * 0.80 * 0.67 / (0.80 + 0.67) ≈ 0.73
```

Η ακρίβεια φαίνεται υψηλή, αλλά η ανάκληση δείχνει ότι το μοντέλο χάνει περίπου το ένα τρίτο των πραγματικά θετικών παραδειγμάτων.

## Δ.10 Μετρικές παλινδρόμησης

Στην παλινδρόμηση, το μοντέλο προβλέπει αριθμητική τιμή. Το σφάλμα είναι η διαφορά ανάμεσα στην πραγματική και την προβλεπόμενη τιμή.

Αν y_i είναι η πραγματική τιμή και ŷ\_i η πρόβλεψη, τότε:

```text
error_i = y_i - ŷ_i
```

### MAE

Το mean absolute error είναι ο μέσος όρος των απόλυτων σφαλμάτων:

```text
MAE = mean(|y_i - ŷ_i|)
```

Είναι εύκολο να ερμηνευθεί, επειδή έχει την ίδια μονάδα με τη μεταβλητή στόχο.

### MSE

Το mean squared error είναι ο μέσος όρος των τετραγώνων των σφαλμάτων:

```text
MSE = mean((y_i - ŷ_i)^2)
```

Τιμωρεί περισσότερο τα μεγάλα λάθη.

### RMSE

Το root mean squared error είναι η τετραγωνική ρίζα του MSE:

```text
RMSE = sqrt(MSE)
```

Έχει την ίδια μονάδα με τη μεταβλητή στόχο και διατηρεί την ισχυρή τιμωρία μεγάλων σφαλμάτων.

### R²

Το R² εκφράζει πόσο μέρος της διακύμανσης της μεταβλητής στόχου εξηγεί το μοντέλο.

```text
R² = 1 - SS_res / SS_tot
```

Τιμή κοντά στο 1 δείχνει καλύτερη προσαρμογή. Τιμή κοντά στο 0 σημαίνει ότι το μοντέλο δεν είναι καλύτερο από μια απλή πρόβλεψη της μέσης τιμής.

Το R² δεν πρέπει να ερμηνεύεται μόνο του. Ένα υψηλό R² δεν σημαίνει απαραίτητα ότι το μοντέλο γενικεύει καλά.

## Δ.11 Μετρικές συμφωνίας επισημειωτών

Όταν πολλά άτομα επισημειώνουν τα ίδια δεδομένα, θέλουμε να μετρήσουμε πόσο συμφωνούν.

### Ποσοστιαία συμφωνία

Η ποσοστιαία συμφωνία είναι το ποσοστό παραδειγμάτων στα οποία οι επισημειωτές δίνουν την ίδια ετικέτα.

```text
agreement = πλήθος συμφωνιών / πλήθος παραδειγμάτων
```

Είναι απλή και κατανοητή, αλλά δεν διορθώνει για τη συμφωνία που μπορεί να προκύψει τυχαία.

### Κάπα κατά Cohen

Το κάπα κατά Cohen μετρά τη συμφωνία δύο επισημειωτών διορθώνοντας για τυχαία συμφωνία.

```text
kappa = (p_o - p_e) / (1 - p_e)
```

όπου:

- p_o είναι η παρατηρούμενη συμφωνία,
- p_e είναι η αναμενόμενη συμφωνία από τύχη.

Το κάπα είναι χρήσιμο, αλλά χρειάζεται προσοχή στην ερμηνεία του. Μπορεί να επηρεαστεί από ανισορροπία των κατηγοριών και από τη συχνότητα κάθε ετικέτας.

## Δ.12 Μετρικές ομαδοποίησης

Στην ομαδοποίηση δεν έχουμε πάντα γνωστές ετικέτες. Γι' αυτό υπάρχουν εσωτερικές και εξωτερικές μετρικές.

### Εσωτερικές μετρικές

Οι εσωτερικές μετρικές αξιολογούν την ομαδοποίηση μόνο με βάση τα δεδομένα και τις ομάδες που βρέθηκαν.

### Silhouette

Το Silhouette συγκρίνει:

- πόσο κοντά είναι ένα σημείο στα σημεία της δικής του ομάδας,
- πόσο κοντά είναι στα σημεία της κοντινότερης άλλης ομάδας.

Παίρνει τιμές από -1 έως 1.

|        Τιμή | Ερμηνεία                                         |
|------------:|--------------------------------------------------|
| Κοντά στο 1 | Το σημείο ταιριάζει καλά στην ομάδα του.         |
| Κοντά στο 0 | Το σημείο βρίσκεται κοντά σε όριο μεταξύ ομάδων. |
|  Κάτω από 0 | Το σημείο ίσως ταιριάζει καλύτερα σε άλλη ομάδα. |

Το Silhouette είναι χρήσιμο, αλλά δεν πρέπει να είναι το μοναδικό κριτήριο επιλογής αριθμού ομάδων.

### Μέθοδος του αγκώνα

Η μέθοδος του αγκώνα εξετάζει πώς αλλάζει μια μετρική καθώς αυξάνεται ο αριθμός ομάδων. Αναζητούμε ένα σημείο όπου η βελτίωση αρχίζει να μειώνεται αισθητά.

Η μέθοδος είναι ευρετική. Δεν εγγυάται ότι υπάρχει σαφές ή μοναδικό σωστό k. 

Καλύτερη μέθοδος θεωρείται η μέθοδος Silhouette, όπου ο αριθμός των ομάδων επιλέγεται με βάση το υψηλότερο Silhouette score.

## Δ.13 Εξωτερικές μετρικές ομαδοποίησης

Οι εξωτερικές μετρικές συγκρίνουν τις ομάδες με γνωστές ετικέτες αναφοράς. Χρησιμοποιούνται όταν υπάρχει κάποιο ground truth.

### NMI

Το normalized mutual information συγκρίνει την πληροφορία που μοιράζονται οι ομάδες και οι πραγματικές ετικέτες.

Παίρνει συνήθως τιμές από 0 έως 1:

- 0: καμία αντιστοιχία,
- 1: τέλεια αντιστοιχία.

Το NMI είναι χρήσιμο όταν οι αριθμοί ή τα ονόματα των ομάδων δεν έχουν άμεση αντιστοίχιση με τις ετικέτες.

### ARI

Το adjusted Rand index συγκρίνει ζεύγη παραδειγμάτων: αν δύο παραδείγματα είναι στην ίδια ή σε διαφορετική ομάδα, και αν αυτό συμφωνεί με τις ετικέτες αναφοράς.

Η διόρθωση για τυχαία συμφωνία κάνει το ARI πιο αυστηρό από απλές μετρήσεις αντιστοίχισης.

## Δ.14 Μετρικές πιθανοτήτων και αβεβαιότητας

Όταν ένα μοντέλο δίνει πιθανότητες, δεν μας ενδιαφέρει μόνο αν η τελική κατηγορία είναι σωστή. Μας ενδιαφέρει και αν οι πιθανότητες είναι αξιόπιστες.

### Log loss

Το log loss τιμωρεί τις βέβαιες αλλά λανθασμένες προβλέψεις:

```text
loss = -log(p)
```

όπου p είναι η πιθανότητα που έδωσε το μοντέλο στη σωστή κατηγορία.

Αν το μοντέλο δώσει πιθανότητα 0.99 σε λάθος κατηγορία, η τιμωρία είναι μεγάλη.

### Brier score

Το Brier score μετρά το τετραγωνικό σφάλμα των προβλεπόμενων πιθανοτήτων.

Για δυαδική ταξινόμηση:

```text
Brier = mean((p_i - y_i)^2)
```

όπου p_i είναι η προβλεπόμενη πιθανότητα της θετικής κατηγορίας και y_i είναι 0 ή 1.

Χαμηλότερη τιμή σημαίνει καλύτερη πιθανοτική πρόβλεψη.

### Calibration

Ένα μοντέλο είναι καλά βαθμονομημένο όταν οι πιθανότητες που δίνει αντιστοιχούν σε πραγματικές συχνότητες.

Αν από όλα τα παραδείγματα στα οποία το μοντέλο δίνει πιθανότητα 0.8, περίπου το 80% είναι σωστά, τότε το μοντέλο είναι καλά βαθμονομημένο σε αυτή την περιοχή.

## Δ.15 Κάλυψη σε συμμορφωτική πρόβλεψη

Στη συμμορφωτική πρόβλεψη, το μοντέλο δεν δίνει μόνο μία ετικέτα ή τιμή. Δίνει ένα σύνολο ή διάστημα που επιδιώκει να περιέχει τη σωστή απάντηση με προκαθορισμένη πιθανότητα.

Η κάλυψη είναι το ποσοστό περιπτώσεων όπου το σύνολο ή διάστημα πρόβλεψης περιέχει τη σωστή τιμή.

```text
coverage = σωστές καλύψεις / σύνολο παραδειγμάτων
```

Αν ζητήσουμε κάλυψη 95%, θέλουμε περίπου 95 στα 100 διαστήματα ή σύνολα πρόβλεψης να περιέχουν τη σωστή απάντηση.

Εκτός από την κάλυψη, εξετάζουμε και το μέγεθος των συνόλων ή διαστημάτων. Πολύ μεγάλα σύνολα έχουν καλή κάλυψη αλλά μικρή χρησιμότητα.

## Δ.16 Πότε χρησιμοποιούμε ποια μετρική

| Περίπτωση                        | Χρήσιμες μετρικές                             |
|----------------------------------|-----------------------------------------------|
| Ισορροπημένη ταξινόμηση          | accuracy, macro-F1                            |
| Ανισόρροπη ταξινόμηση            | macro-F1, precision, recall, ανά κατηγορία F1 |
| Υψηλό κόστος ψευδώς θετικών      | precision                                     |
| Υψηλό κόστος ψευδώς αρνητικών    | recall                                        |
| Παλινδρόμηση                     | MAE, RMSE, R²                                 |
| Επισημείωση από δύο κριτές       | ποσοστιαία συμφωνία, κάπα του Cohen           |
| Ομαδοποίηση χωρίς ετικέτες       | Silhouette, οπτική επιθεώρηση                 |
| Ομαδοποίηση με ετικέτες αναφοράς | NMI, ARI                                      |
| Πιθανοτικές προβλέψεις           | log loss, Brier score, calibration            |
| Συμμορφωτική πρόβλεψη            | coverage, μέγεθος συνόλου ή διαστήματος       |

## Δ.17 Συχνές παγίδες

### Μόνο μία μετρική

Η χρήση μίας μόνο μετρικής μπορεί να κρύψει σημαντικά προβλήματα. Είναι καλύτερο να εξετάζουμε περισσότερες από μία, ειδικά σε ανισόρροπα δεδομένα.

### Accuracy σε ανισόρροπες κατηγορίες

Υψηλή accuracy μπορεί να προκύψει ακόμη κι αν το μοντέλο αγνοεί σπάνιες αλλά σημαντικές κατηγορίες.

### Macro και micro χωρίς ερμηνεία

Το macro-F1 και το micro-F1 απαντούν σε διαφορετικές ερωτήσεις. Το πρώτο δίνει ίσο βάρος στις κατηγορίες, ενώ το δεύτερο επηρεάζεται περισσότερο από τις συχνές κατηγορίες.

### Σύγκριση μετρικών σε διαφορετικά datasets

Οι μετρικές δεν είναι πάντα άμεσα συγκρίσιμες ανάμεσα σε διαφορετικά datasets, ειδικά όταν αλλάζει η κατανομή των κατηγοριών ή η δυσκολία του προβλήματος.

### Μετρικές χωρίς ανάλυση λαθών

Μια μετρική λέει πόσο καλά ή άσχημα πήγε ένα μοντέλο, αλλά όχι πάντα γιατί. Η ανάλυση λαθών παραμένει απαραίτητη.

## Δ.18 Ελάχιστη αναφορά αποτελεσμάτων

Όταν παρουσιάζουμε αποτελέσματα, είναι χρήσιμο να αναφέρουμε:

- το dataset,
- το πλήθος παραδειγμάτων,
- την κατανομή κατηγοριών,
- τον τρόπο διαχωρισμού train/test,
- τις μετρικές που χρησιμοποιήθηκαν,
- αν οι μετρικές είναι macro, micro ή weighted,
- τον πίνακα σύγχυσης όπου είναι χρήσιμος,
- μέση τιμή και τυπική απόκλιση αν έγιναν πολλαπλές εκτελέσεις,
- σύντομη ανάλυση λαθών.

Η αξιολόγηση δεν είναι απλώς αριθμητική διαδικασία. Είναι τρόπος να καταλάβουμε τι έμαθε το μοντέλο, πού αποτυγχάνει και πόσο μπορούμε να εμπιστευτούμε τα αποτελέσματά του.
