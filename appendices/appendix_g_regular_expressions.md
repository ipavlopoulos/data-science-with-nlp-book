# Παράρτημα Ζ: Κανονικές εκφράσεις

Οι κανονικές εκφράσεις, ή regular expressions, είναι ένας τρόπος περιγραφής μοτίβων μέσα σε κείμενο. Χρησιμοποιούνται για αναζήτηση, καθαρισμό, εξαγωγή πληροφορίας και έλεγχο μορφής.

Το παράρτημα αυτό λειτουργεί ως σύντομη αναφορά. Τα παραδείγματα χρησιμοποιούν τη βιβλιοθήκη `re` της Python.

## Ζ.1 Βασική χρήση στην Python

Η βιβλιοθήκη `re` παρέχει συναρτήσεις για αναζήτηση και αντικατάσταση μοτίβων.

```python
import re

text = "Η απόφαση έχει αριθμό 12345."
match = re.search(r"\d+", text)

if match:
    print(match.group())
```

Αποτέλεσμα:

```text
12345
```

Συνηθισμένες συναρτήσεις:

| Συνάρτηση | Χρήση |
|---|---|
| `re.search()` | Βρίσκει την πρώτη εμφάνιση μοτίβου. |
| `re.findall()` | Επιστρέφει όλες τις εμφανίσεις. |
| `re.finditer()` | Επιστρέφει επαναλήπτη με match objects. |
| `re.sub()` | Αντικαθιστά μοτίβα. |
| `re.split()` | Χωρίζει κείμενο με βάση μοτίβο. |
| `re.fullmatch()` | Ελέγχει αν όλο το κείμενο ταιριάζει στο μοτίβο. |

## Ζ.2 Raw strings

Στην Python, τα regex γράφονται συνήθως ως raw strings, με πρόθεμα `r`:

```python
pattern = r"\d+"
```

Αυτό αποφεύγει συγκρούσεις ανάμεσα στους χαρακτήρες διαφυγής της Python και στους χαρακτήρες διαφυγής των regex.

Χωρίς raw string, το `\n`, το `\t` και άλλες ακολουθίες μπορεί να ερμηνευθούν πρώτα από την Python.

## Ζ.3 Κυριολεκτικοί χαρακτήρες

Οι απλοί χαρακτήρες ταιριάζουν με τον εαυτό τους:

```python
re.findall(r"μέρα", "καλή μέρα, δύσκολη μέρα")
```

Αποτέλεσμα:

```text
["μέρα", "μέρα"]
```

Αν θέλουμε να αναζητήσουμε χαρακτήρα που έχει ειδική σημασία σε regex, τον διαφεύγουμε με `\`.

Παράδειγμα:

```python
re.findall(r"\.", "Τέλος. Αρχή.")
```

Το `.` μόνο του σημαίνει «οποιοσδήποτε χαρακτήρας», ενώ το `\.` σημαίνει κυριολεκτική τελεία.

## Ζ.4 Μεταχαρακτήρες

Βασικοί μεταχαρακτήρες:

| Μοτίβο | Σημασία |
|---|---|
| `.` | Οποιοσδήποτε χαρακτήρας εκτός από αλλαγή γραμμής. |
| `^` | Αρχή γραμμής ή κειμένου. |
| `$` | Τέλος γραμμής ή κειμένου. |
| `*` | Μηδέν ή περισσότερες επαναλήψεις. |
| `+` | Μία ή περισσότερες επαναλήψεις. |
| `?` | Μηδέν ή μία επανάληψη. |
| `{m}` | Ακριβώς `m` επαναλήψεις. |
| `{m,n}` | Από `m` έως `n` επαναλήψεις. |
| `[]` | Κλάση χαρακτήρων. |
| `()` | Ομάδα. |
| `|` | Εναλλακτική. |

## Ζ.5 Κλάσεις χαρακτήρων

Μια κλάση χαρακτήρων ταιριάζει με έναν χαρακτήρα από ένα σύνολο.

```python
re.findall(r"[αο]πό", "από οπό υπό")
```

Παράδειγμα για ελληνικά πεζά:

```python
re.findall(r"[α-ωάέήίόύώ]+", "Καλή μέρα!")
```

Το παραπάνω δεν πιάνει κεφαλαία. Για κεφαλαία και πεζά:

```python
re.findall(r"[Α-Ωα-ωΆΈΉΊΌΎΏάέήίόύώ]+", "Καλή Μέρα!")
```

Αρνητική κλάση:

```python
re.findall(r"[^,]+", "Αθήνα,Πάτρα,Ηράκλειο")
```

Το `[^,]` σημαίνει «οποιοσδήποτε χαρακτήρας εκτός από κόμμα».

## Ζ.6 Συντομογραφίες χαρακτήρων

Συνηθισμένες συντομογραφίες:

| Μοτίβο | Σημασία |
|---|---|
| `\d` | Ψηφίο. |
| `\D` | Μη ψηφίο. |
| `\w` | Χαρακτήρας λέξης. |
| `\W` | Μη χαρακτήρας λέξης. |
| `\s` | Κενό, tab ή αλλαγή γραμμής. |
| `\S` | Μη κενός χαρακτήρας. |

Προσοχή: η σημασία του `\w` μπορεί να εξαρτάται από Unicode συμπεριφορά. Για ελληνικό κείμενο, συχνά είναι πιο καθαρό να ορίζουμε ρητά τις κλάσεις χαρακτήρων που θέλουμε.

## Ζ.7 Επανάληψη

Παραδείγματα:

```python
re.findall(r"\d+", "12 μήνες και 365 μέρες")
```

Αποτέλεσμα:

```text
["12", "365"]
```

Ακριβώς τέσσερα ψηφία:

```python
re.findall(r"\b\d{4}\b", "Το έτος 2024 και ο αριθμός 12345")
```

Από δύο έως τέσσερα ψηφία:

```python
re.findall(r"\b\d{2,4}\b", "7 12 123 1234 12345")
```

## Ζ.8 Ομάδες

Οι παρενθέσεις δημιουργούν ομάδες:

```python
text = "Απόφαση 123/2024"
match = re.search(r"(\d+)/(\d{4})", text)

if match:
    print(match.group(1))
    print(match.group(2))
```

Αποτέλεσμα:

```text
123
2024
```

Οι ομάδες είναι χρήσιμες όταν θέλουμε να εξαγάγουμε συγκεκριμένα τμήματα ενός μοτίβου.

## Ζ.9 Εναλλακτικές

Το `|` δηλώνει εναλλακτική:

```python
re.findall(r"γάτα|σκύλος", "η γάτα και ο σκύλος")
```

Με ομάδα:

```python
re.findall(r"(Αθήνα|Πάτρα|Ηράκλειο)", "Αθήνα και Πάτρα")
```

Αν δεν θέλουμε να κρατήσουμε την ομάδα ως captured group, χρησιμοποιούμε non-capturing group:

```python
re.findall(r"(?:Αθήνα|Πάτρα|Ηράκλειο)", "Αθήνα και Πάτρα")
```

## Ζ.10 Αρχή και τέλος

Το `^` ταιριάζει στην αρχή και το `$` στο τέλος:

```python
re.search(r"^Απόφαση", "Απόφαση 123/2024")
```

```python
re.search(r"2024$", "Απόφαση 123/2024")
```

Για έλεγχο ολόκληρης συμβολοσειράς, συχνά είναι πιο καθαρό το `re.fullmatch()`:

```python
re.fullmatch(r"\d{4}", "2024")
```

## Ζ.11 Όρια λέξης

Το `\b` δηλώνει όριο λέξης:

```python
re.findall(r"\bκαι\b", "καιρός και κείμενο")
```

Αποτέλεσμα:

```text
["και"]
```

Προσοχή στα ελληνικά και στο Unicode: σε ορισμένες περιπτώσεις, ειδικά με σημεία στίξης ή διακριτικά, είναι καλύτερο να ελέγχουμε εμπειρικά αν το `\b` συμπεριφέρεται όπως περιμένουμε.

## Ζ.12 Άπληστη και μη άπληστη αντιστοίχιση

Οι ποσοδείκτες είναι άπληστοι από προεπιλογή: προσπαθούν να ταιριάξουν όσο περισσότερο γίνεται.

```python
text = "<b>πρώτο</b><b>δεύτερο</b>"
re.findall(r"<b>.*</b>", text)
```

Το `.*` μπορεί να πιάσει από το πρώτο `<b>` έως το τελευταίο `</b>`.

Μη άπληστη μορφή:

```python
re.findall(r"<b>.*?</b>", text)
```

Η μη άπληστη μορφή σταματά στο πρώτο δυνατό ταίριασμα.

Για πραγματική ανάλυση HTML, όμως, προτιμάμε parser όπως το BeautifulSoup.

## Ζ.13 Lookahead και lookbehind

Το lookahead ελέγχει τι ακολουθεί χωρίς να το καταναλώνει:

```python
re.findall(r"\d+(?= ευρώ)", "10 ευρώ και 20 δολάρια")
```

Αποτέλεσμα:

```text
["10"]
```

Το lookbehind ελέγχει τι προηγείται:

```python
re.findall(r"(?<=ΑΔΑ: )\w+", "ΑΔΑ: Ψ123ABC")
```

Τα lookarounds είναι χρήσιμα, αλλά μπορούν να κάνουν τα regex δυσανάγνωστα. Χρησιμοποιούνται όταν πραγματικά χρειάζεται το συμφραζόμενο χωρίς να συμπεριληφθεί στο αποτέλεσμα.

## Ζ.14 Αντικατάσταση με re.sub

Η `re.sub()` αντικαθιστά μοτίβα:

```python
text = "καλή    μέρα"
clean = re.sub(r"\s+", " ", text).strip()
```

Αφαίρεση αριθμών:

```python
re.sub(r"\d+", "", "Απόφαση 123/2024")
```

Αντικατάσταση με ομάδες:

```python
text = "2024-05-23"
re.sub(r"(\d{4})-(\d{2})-(\d{2})", r"\3/\2/\1", text)
```

Αποτέλεσμα:

```text
23/05/2024
```

## Ζ.15 Διάσπαση με re.split

Η `re.split()` χωρίζει κείμενο με βάση μοτίβο:

```python
text = "Αθήνα, Πάτρα; Ηράκλειο"
re.split(r"[,;]\s*", text)
```

Αποτέλεσμα:

```text
["Αθήνα", "Πάτρα", "Ηράκλειο"]
```

## Ζ.16 Χρήσιμα μοτίβα για ελληνικό κείμενο

Ελληνικές λέξεις με μονοτονικά γράμματα:

```python
r"[Α-Ωα-ωΆΈΉΊΌΎΏάέήίόύώ]+"
```

Ελληνικές λέξεις και τελικό σίγμα:

```python
r"[Α-Ωα-ωΆΈΉΊΌΎΏάέήίόύώς]+"
```

Ακολουθίες αριθμών:

```python
r"\d+"
```

Έτος με τέσσερα ψηφία:

```python
r"\b\d{4}\b"
```

Πολλαπλά κενά:

```python
r"\s+"
```

URL απλής μορφής:

```python
r"https?://\S+"
```

Email απλής μορφής:

```python
r"[\w.+-]+@[\w-]+\.[\w.-]+"
```

Τα μοτίβα για URLs και emails δεν καλύπτουν όλες τις πιθανές έγκυρες περιπτώσεις. Είναι πρακτικά μοτίβα για καθαρισμό ή αρχική εξαγωγή, όχι πλήρεις επικυρωτές.

## Ζ.17 Παράδειγμα: εξαγωγή χρονολογιών

```python
text = "Το 1821 και το 2024 είναι δύο χρονολογίες."
years = re.findall(r"\b\d{4}\b", text)
```

Αποτέλεσμα:

```text
["1821", "2024"]
```

Αν θέλουμε να περιορίσουμε σε έτη που αρχίζουν από 18, 19 ή 20:

```python
years = re.findall(r"\b(?:18|19|20)\d{2}\b", text)
```

## Ζ.18 Παράδειγμα: καθαρισμός κενών

```python
text = "  καλή     μέρα \n σε όλους  "
clean = re.sub(r"\s+", " ", text).strip()
```

Αποτέλεσμα:

```text
καλή μέρα σε όλους
```

## Ζ.19 Παράδειγμα: εξαγωγή πληροφορίας από κείμενο

Έστω κείμενο:

```text
Πόλη: Αθήνα, Πληθυσμός: 643452
```

Μπορούμε να εξαγάγουμε πόλη και πληθυσμό:

```python
text = "Πόλη: Αθήνα, Πληθυσμός: 643452"
match = re.search(r"Πόλη:\s*([^,]+),\s*Πληθυσμός:\s*(\d+)", text)

if match:
    city = match.group(1)
    population = int(match.group(2))
```

## Ζ.20 Regex και pandas

Η `pandas` επιτρέπει χρήση regex σε κειμενικές στήλες:

```python
df["has_year"] = df["text"].str.contains(r"\b\d{4}\b", regex=True)
```

Εξαγωγή ομάδας:

```python
df["year"] = df["text"].str.extract(r"(\b\d{4}\b)")
```

Αντικατάσταση:

```python
df["text_clean"] = df["text"].str.replace(r"\s+", " ", regex=True)
```

Προσοχή στις κενές τιμές. Συχνά χρειάζεται:

```python
df["text"].fillna("").str.contains(r"\b\d{4}\b", regex=True)
```

## Ζ.21 Regex, HTML και JSON

Οι κανονικές εκφράσεις είναι χρήσιμες για μικρά μοτίβα μέσα σε κείμενο. Δεν είναι όμως κατάλληλες ως γενικός parser για δομημένες μορφές όπως HTML ή JSON.

Για HTML, προτιμάμε:

```python
from bs4 import BeautifulSoup
```

Για JSON, προτιμάμε:

```python
import json
```

Τα regex μπορούν να χρησιμοποιηθούν αφού έχουμε εντοπίσει το σωστό στοιχείο ή όταν θέλουμε να καθαρίσουμε μικρά αποσπάσματα.

## Ζ.22 Αποσφαλμάτωση regex

Όταν ένα regex δεν δουλεύει όπως περιμένουμε:

1. Ξεκινάμε με μικρό παράδειγμα κειμένου.
2. Δοκιμάζουμε το μοτίβο βήμα προς βήμα.
3. Ελέγχουμε αν χρειάζεται raw string.
4. Ελέγχουμε αν κάποιος χαρακτήρας έχει ειδική σημασία.
5. Ελέγχουμε την απληστία των ποσοδεικτών.
6. Ελέγχουμε Unicode, τόνους και κεφαλαία.
7. Χρησιμοποιούμε `findall()` ή `finditer()` για να δούμε τι ταιριάζει.

Παράδειγμα:

```python
for match in re.finditer(r"\b\d{4}\b", text):
    print(match.group(), match.start(), match.end())
```

## Ζ.23 Συχνές παγίδες

### Ξεχασμένο raw string

Προτιμάμε:

```python
r"\d+"
```

αντί για:

```python
"\d+"
```

### Άπληστο `.*`

Το `.*` μπορεί να ταιριάξει πολύ μεγαλύτερο τμήμα από όσο περιμένουμε. Δοκιμάζουμε `.*?` ή πιο συγκεκριμένο μοτίβο.

### Υπερβολικά γενικά μοτίβα

Το μοτίβο:

```python
r".+"
```

ταιριάζει σχεδόν τα πάντα. Συνήθως χρειαζόμαστε πιο περιορισμένο μοτίβο.

### Αφαίρεση χρήσιμης πληροφορίας

Αν αφαιρέσουμε όλους τους τόνους, όλα τα σημεία στίξης ή όλους τους αριθμούς, μπορεί να χάσουμε χρήσιμο σήμα.

### Regex αντί για parser

Για HTML και JSON, χρησιμοποιούμε parser. Τα regex είναι βοηθητικά, όχι γενική λύση δομημένης ανάλυσης.

## Ζ.24 Μικρό cheat sheet

| Θέλω να βρω | Μοτίβο |
|---|---|
| Ψηφία | `\d+` |
| Έτος | `\b\d{4}\b` |
| Κενά | `\s+` |
| Ελληνικές λέξεις | `[Α-Ωα-ωΆΈΉΊΌΎΏάέήίόύώς]+` |
| Οτιδήποτε μέχρι κόμμα | `[^,]+` |
| Αρχή κειμένου | `^` |
| Τέλος κειμένου | `$` |
| Μία ή περισσότερες επαναλήψεις | `+` |
| Μηδέν ή περισσότερες επαναλήψεις | `*` |
| Προαιρετικό στοιχείο | `?` |
| Ομάδα | `(...)` |
| Εναλλακτική | `a|b` |
| Μη άπληστη επανάληψη | `*?` |

## Ζ.25 Λίστα ελέγχου

Πριν χρησιμοποιήσουμε regex σε ανάλυση δεδομένων, ελέγχουμε:

- Έχουμε μικρά παραδείγματα όπου ξέρουμε το σωστό αποτέλεσμα;
- Χρησιμοποιούμε raw string;
- Έχουμε διαφύγει ειδικούς χαρακτήρες;
- Χρειάζεται να χειριστούμε ελληνικούς τόνους;
- Χρειάζεται να χειριστούμε κεφαλαία και πεζά;
- Είναι το μοτίβο υπερβολικά γενικό;
- Είναι η απληστία των ποσοδεικτών σωστή;
- Χρησιμοποιούμε parser όταν η είσοδος είναι HTML ή JSON;
- Έχουμε ελέγξει τι συμβαίνει με κενές τιμές;
- Έχουμε τεκμηριώσει τι καθαρίζουμε και γιατί;

Οι κανονικές εκφράσεις είναι ισχυρό εργαλείο, αλλά χρειάζονται προσοχή. Ένα μικρό λάθος σε regex μπορεί να αφαιρέσει δεδομένα, να κρατήσει λάθος μοτίβα ή να δημιουργήσει σιωπηλά σφάλματα στην ανάλυση.
